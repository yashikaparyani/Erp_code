FRS (Phase 1)
Integrated Tender-to-Delivery ERP (Govt Projects, Multi-site, O&M ready)
1) Purpose (Phase 1)
Build a role-based system to manage the core lifecycle:
Tender → Survey → BOQ → Costing → Approval → Procurement → Delivery → Installation → Commissioning → Billing → Payment tracking → Ticketing basics
Phase 1 explicitly excludes:
External accounting ERP integration (Tally/Zoho/SAP posting)
AI/predictive analytics
Biometric integration
2) Users & Roles (Phase 1)
The system shall support these roles (minimum):
Superadmin 
Department Head
Presales/Tendering
Engineering Head + Engineers
Purchase
Stores/Logistics (HO & Project)
Project Manager
Accounts
Field Technician
O&M Operator
Top Management
Core rules:
A user can have multiple roles (only via Superadmin permission management).
Role access is scoped (project + site + region), but Phase 1 enforcement will be project-first (site/region optional scoping can still exist in data).
3) Cross-cutting Requirements (Phase 1)
3.1 Audit & Approval
All critical business entities must have:
created_by, created_at, updated_by, updated_at
status + status_history (or audit events)
Approval workflow shall be supported for: Tender, Costing, Indent, PO, GRN/Stock movements, Invoices, Documents, Commissioning.
Approvals are not value-threshold based (same flow regardless of amount).
Approved records editable only by Superadmin/Department Head, and must create an audit entry.
3.2 Document Management
DMS is mandatory in Phase 1.
Documents must be linked to Tender/Project/Site/PO/GRN/Commissioning etc.
Manual versioning (v1/v2/…)
Expiry tracking + alerts (insurance, warranty certificates, BG expiry etc.)
Old versions can be deleted by Dept Head (must be audited).
3.3 Dashboards (Phase 1)Phase 1 must deliver role dashboards:
Project Head: indents pending, milestone billing upcoming, payment pending, dependencies, SLA risk, manpower shortage, red flags
Presales: active tenders, submission deadlines, BOQ readiness, survey status, compliance checklist, EMD/PBG tracker
Procurement: indents, comparison sheets, 3-quotation compliance, PO approval queue, delivery tracking,payment dues, advance requirements, PDC requirements
Stores: GRN queue, dispatch, stock position, aging,consumption(project wise and site wise)
Execution: timeline vs actual, DPR, site photos, dependencies, manpower deployed
Accounts: billing tracker, payment aging, retention, GST/TDS entry, penalty deductions at payment stage
O&M: ticket queue + SLA timer basics (Phase 1 minimum), RMA tracker
4) Module Functional Requirements (Phase 1)
4.1 Tender & Presales
Create/edit tenders: tender no, client, submission date, status
Attach RFP documents
Maintain compliance checklist items (line-level status + remarks)
Maintain clarification tracker (question/answer dates)
Track EMD/PBG instruments: bank, instrument no, amount, issue/expiry, status
Convert tender to project only when won
4.2 Survey
Record survey per tender/project and per site
Survey must be marked “completed” before BOQ can be approved
Survey photos and site notes upload
4.3 BOQ & Costing
BOQ is site-wise, and aggregated to project level
BOQ supports revisions (versioning), retain history for long period
Costing is margin-on-cost
BOQ approval and costing approval via workflow
BOQ lines can be linked to Item Master, but allow ad-hoc description lines
4.4 Procurement & Vendor
Vendor master management
Indent creation per project, optionally site-specific
Mandatory 3-quotation compliance, with exception approval for single-supplier scenarios
Quotation comparison sheet generation
PO creation:
One PO can cover multiple indents
Indent lines can split across multiple POs/vendors
PO amendments allowed, must be tracked as revisions
4.5 Stores & Logistics
GRN:
Partial deliveries supported
Accept/reject quantities
Attach packing list / warranty certificate / MIR (Material Inspection Report)
Dispatch:
HO → Project store → Site OR Vendor → Site direct delivery (both supported)
Inventory rules:
Negative stock not allowed
Ledger-based movements for audit
Serialized tracking for critical devices (IP devices at minimum)
4.6 Project Execution + Dependency Engine
Create project with project type template (ITMS/Surveillance/Solar/O&M)
Maintain sites per project (site status)
Milestones and tasks:
planned vs actual dates
owner assignment
Dependency engine:
restrict progression if prerequisites incomplete (survey, drawings approved, material delivered, IP assigned, etc.)
show blocking reason
override allowed only by Dept Head+ with written justification and audit
DPR:
daily per site
photos + summary
4.7 Engineering & Design (Phase 1 scope)
Track engineering outputs and approvals:
Drawing preparation status + revision history + client approval status
Technical deviation log + change request tracker
Engineering deliverables list includes drawings/layouts, network plans, design reports, FAT/SAT/UAT artefacts etc. 
4.8 Network & Commissioning
Device register per site (serial-based)
IP pool and allocation log per project/site
Commissioning checklist completion
Test report upload
Camera FOV and Commissioning complete requires client signoff
4.9 Billing & Accounts (Tracking only in Phase 1)
Invoice types: Milestone / RA / O&M
Invoice can be raised before milestone completion (flag + audit)
Payment receipt tracking:
payment date, amount received, TDS amount, remarks
Retention ledger per project (retention % is project-wise)
Penalties applied at payment stage (store as deduction records)
4.10 O&M + Ticketing (Phase 1 minimum)
Ticket creation linked to project/site/asset if available
SLA timer:
24X7 365 days
pause/resume with role restrictions
Penalty model stored as per-ticket and monthly aggregate rules (calculation engine can be “basic” in Phase 1, richer in Phase 2)
RMA tracker
4.11 Alerts (Phase 1 basic)
In-app notifications for:
milestone due reminders (default 15 days or configured)
payment overdue (configurable)
dependency blocks
document expiry reminders
Escalation chain: hierarchical upward (configurable)
